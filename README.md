# ContextSOP

ContextSOP turns raw, messy incident transcripts and system engineering logs into safe, structured, and interactive Standard Operating Procedures (SOPs). By parsing inputs using LLMs, it outputs a strict JSON DSL that renders step-by-step checklists, parameterized commands, and automated checks.

For the complete architectural specifications and phase-by-phase implementation blueprint, refer to the [Implementation Document](sources/document.md).

## System Architecture

The following diagram illustrates the flow from a raw log or incident transcript to an interactive runbook page:

```
[Raw Transcript/Log]
        │
        ▼ (POST /api/v1/sop/generate)
┌──────────────────────────────────────┐
│  LLM Parsing Engine (gpt-4o-mini)    │
│  - Redacts sensitive credential tokens│
│  - Structurizes tasks to JSON DSL    │
└──────────────────────────────────────┘
        │
        ▼ (Pydantic DAG Schema Validation)
┌──────────────────────────────────────┐
│  Workflow DSL Checkers & Typeguards  │
│  - Verifies unique variable maps     │
│  - Checks cycles and dependencies    │
└──────────────────────────────────────┘
        │
        ▼ (Supabase / local DB sync)
┌──────────────────────────────────────┐
│  Interactive Runbook Engine          │
│  - Parameter overrides interpolation │
│  - Isolated components sandbox       │
│  - SSRF-mitigated URL verifications  │
└──────────────────────────────────────┘
```

## Key Security Safeguards

1. **Execution Isolation Sandbox**
   Custom UI components generated by the AI model are rendered using an `iframe` configured with strict sandbox options: `sandbox="allow-scripts"`. Omitting `allow-same-origin` ensures that executing scripts cannot access host domain cookies or `localStorage` data.
2. **SSRF Mitigation**
   Automated verification checks are proxied through the backend (`POST /api/v1/sop/verify`). Target URLs must match whitelisted external domains (`github.com`, `api.github.com`, `google.com`, `httpbin.org`, `status.payment-service.com`) or subdomains ending in `.status.payment-service.com`. Attempts to call internal resources are blocked.
3. **Clipboard Injection Prevention**
   Copy-to-clipboard handlers remove carriage return characters (`\r`) and leading/trailing whitespace/newlines to block terminal command injection vectors.

|---|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Endpoint url for the Supabase project | `https://your-project.supabase.co` |

## Local Setup

### 1. Frontend
```bash
cd frontend
npm install --legacy-peer-deps
npm run dev
```

### 2. Backend
```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python run.py
```
